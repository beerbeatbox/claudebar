ClaudeBar — Ring animation concepts

ทุกแบบยึดวงแหวน usage เดิม (r=6, stroke 2.6, เริ่มที่ 12 นาฬิกา) แล้วเพิ่ม motion เป็น layer ตาม"สถานะ" — ไม่ทิ้ง gauge. ซ้าย = ขนาดจริงบน menu bar, ขวา = ซูม ~7×. พื้นหลังหมากรุกไว้เช็คความคมตอนทับ wallpaper.

A · Indeterminate spinner — ตอนกำลัง refresh (แนะนำสำหรับ loading)

ระหว่าง spawn claude -p "/usage" วงแหวนความจริงถูกแทนด้วยส่วนโค้งสั้นๆ หมุนรอบ track (เหมือน Material indeterminate) พอข้อมูลมาถึงค่อย snap กลับเป็น % จริง. สื่อ "กำลังโหลด" ชัด และหยุดเองเมื่อเสร็จ → ประหยัดแบต.

B · Fill-to-value — ตอนได้ข้อมูลใหม่ (แนะนำคู่กับ A)

พอ % เปลี่ยน วงแหวนวิ่งเติมจากค่าเก่า→ค่าใหม่ด้วย easing (ไม่กระโดด). ผูกกับ "usage" ตรงๆ ที่สุด, เห็นชัดว่าค่าขยับขึ้น/ลง. (ในตัวอย่าง loop 0→66% ไปเรื่อยๆ)

C · Breathing pulse — ตอนใกล้ลิมิต (เช่น >90%)

วงแหวน (สีแดง) หายใจเข้า-ออก: opacity + ความหนาเส้นเต้นเบาๆ เป็น "คำเตือน" ที่สังเกตเห็นด้วยหางตา โดยไม่รบกวน. ใช้เฉพาะ threshold สูง ไม่ใช่ตลอดเวลา.

D · Comet highlight — โหมด idle แบบ subtle (ออปชัน)

วงแหวน usage จริงอยู่นิ่ง แต่มีจุดสว่าง (comet) วิ่งวนรอบ ๆ ให้รู้สึกว่าแอป "ยังมีชีวิต". สวยแต่กินแบตถ้าเปิดตลอด — เหมาะเป็นออปชันเปิด/ปิดมากกว่าค่า default.

ระบบที่แนะนำ: A (refresh) + B (เปลี่ยนค่า) เป็นแกนหลัก — ได้ motion ที่ "เป็น usage" 100% และหยุดเองเสมอ. เพิ่ม C ตอน >90% เป็นคำเตือน. D เก็บไว้เป็นออปชัน.
