Eight candidate layouts for the status item, each shown in a live mini menu bar. Drag the sliders to see threshold colors (amber ≥ 75%, red ≥ 90%) and how each layout breathes as the countdown text changes. Width shown is the live rendered width — menu bar real estate is the main trade-off.

Implementation notes. Every variant maps onto the existing tray_manager setup: the glyph keeps rendering through renderTrayGlyph (PNG via the method channel) and the text goes in setTitle. The stacked variant (E) needs the whole item rendered as one image instead of icon + title, since NSStatusItem titles are single-line. Countdown granularity of one minute means a 60s re-render timer in TrayController; rounding to 5-minute steps ("2h 15m") would let it re-render lazily with the usual refresh tick.
